# Cursor rules

These small compatibility notes help older Cursor models navigate gitgud without relying on stale framework or database assumptions.

- [`database.mdc`](database.mdc) points database work to the current schema documentation.
- [`styles.mdc`](styles.mdc) points interface work to the active styling conventions.

Project documentation lives in the [main README](../../README.md). Database changes must also follow the [SQL guide](../../sql/README.md), including its Row Level Security and least-privilege verification steps.

Keep rules short and factual. Put durable project documentation in the repository READMEs rather than duplicating it here.
